Clash Verge Tun 模式配置记录

发表于 2023-11-04 更新于 2023-11-07 分类于 折腾 阅读次数： 本文字数： 3.2k 阅读时长 ≈ 11 分钟
在本文中，我将记录Clash Verge的配置过程，包括创建配置文件、切换内核、启用TUN模式以及防止DNS泄露。

起因

2023年11月2日下午，Clash for Windows（CFW）仓库被删除，当晚相关话题冲上微博热搜前十，吸引来一些又蠢又坏的自媒体煽风点火，谣言满天飞。次日，Clash系的多数仓库均采取了删库或归档切换分支的措施，暂避风头。由于CFW不再更新，出于安全考虑，我决定从CFW切换到Clash Verge。

下载与安装

以下是Clash Verge的GitHub Release页面，请根据你的操作系统选择相应的安装包：

  • Windows用户可以下载 Clash.Verge_1.x.x_x64_zh-CN.msi
  • 使用Intel CPU的Mac用户可以下载 Clash.Verge_1.x.x_x64.dmg
  • 使用M系列CPU的Mac用户可以下载 Clash.Verge_1.x.x_aarch64.dmg

Clash Verge

Clash Verge Release 页面

安装过程非常简单，只需一直点击下一步即可，无需进行其他设置。

基础配置

首先，点击左侧边栏中的Settings以打开设置页面。

设置中文

在Verge Setting中将Language设置为中文。

切换内核

Clash Verge支持两个内核：Clash Premium（闭源且已删库）和Clash Meta（开源）。默认情况下使用的是Clash Premium，建议将内核切换为Clash Meta。

  1. 在Clash设置中找到Clash内核项，然后点击后面的小齿轮图标。
  2. 选择Clash Meta。
  3. 点击右上角的重启内核。

设置外部控制

之前，Clash曝出了一个漏洞，攻击者可以通过RESTful API接口植入木马。出于安全考虑，我修改了默认的外部控制端口并添加了密码。详细信息请看不良林的Youtube视频。

  1. 在Clash设置中找到外部控制项，然后点击后面的箭头。
  2. 修改External Controller的端口，默认是9090，可以改成11451（手动狗头）。
  3. 在Core Secret中填写密码。
  4. 保存。

其他配置

以下是我自己的配置，仅供参考，请根据你的需求进行设置。

  • 禁用IPv6
  • 禁用局域网连接
  • 开启开机启动
  • 开启静默启动

配置文件设置

  1. 点击左侧边栏中的配置。
  2. 点击右上角的新建。
  3. 类型选择Local，名称自定，其余两项留空。
    • 更多配置文件类型，请参考Clash Verge Wiki。
  4. 点击保存。
  5. 右键单击新建的配置项，选择打开文件，默认会使用VSCode打开。
  6. 填写自己的配置并保存。
    • 如果不熟悉配置，可以参考Clash Meta文档，在配置示例中有个懒人配置可以直接拿来用。
    • 也可以直接使用CFW的配置文件大部分内容。

启用TUN模式

TUN接口是一种虚拟网络接口，允许用户空间程序与内核之间的通信，以便将数据包从一个网络堆栈传递到另一个。Clash Meta支持启用TUN模式，它通过创建一个TUN虚拟网卡来接管用户网络中的三层数据包。在TUN模式下，所有应用程序和流量都将被强制路由到代理服务器，无论应用程序是否支持代理设置。

  1. 点击左侧边栏中的设置，找到服务模式并点击后面的小盾牌图标。
  2. 点击INSTALL。
    • 安装时可能会报告安装失败，但实际上是成功的，这可能是一个Bug。
  3. 打开服务模式与Tun模式的开关，同时关闭系统代理的开关。
  4. 如果任务栏左下角的网络图标由WiFi变成下图所示的图标，说明安装成功。
  5. 点击左侧边栏中的配置，右键单击之前创建的配置项，然后选择使用。

防止DNS泄露

DNS泄露（DNS Leak）是一种网络隐私问题，通常发生在使用虚拟专用网络（VPN）或代理服务时。它指的是用户的DNS查询未通过VPN或代理处理，而是绕过它们，直接发送到本地或公共DNS服务器。这可能会泄露用户的在线活动信息，因为DNS查询通常包含用户正在访问的网站信息，即使其网络流量通过VPN或代理加密和隐藏。

有关DNS泄露的更多信息，请参考不良林的相关视频：

  • 进阶•DNS泄露篇
  • 进阶•DNS分流篇
  • 进阶•DNS代理篇

禁用智能多宿主名称解析

Windows系统的智能多宿主域名解析会同时向所有网络发出并行的DNS查询以提高性能。尽管我在配置文件中设置了相关的DNS规则，但这些规则仅对Clash设置的Tun虚拟网卡生效，而物理网卡不受其控制。因此，未禁用智能多宿主名称解析的情况下，系统可能会同时通过虚拟网卡和物理网卡发起DNS查询，而物理网卡使用的DNS服务器一般是本地或公共DNS，也就意味着运营商等第三方可以看到你正在访问的网站，此时就发生了DNS泄露。

这一步可以使用WireShark抓取物理网卡的数据来对比禁用前后的区别。

关闭方法如下：

  1. 在开始菜单搜索编辑组策略并打开对应控制面板。
  2. 打开计算机配置-管理模板-网络-DNS 客户端-禁用智能多宿主名称解析。
  3. 勾选左上角的已启用。
  4. 点击确定。

禁用QUIC

QUIC（Quick UDP Internet Connections）是一种网络传输协议，旨在加速互联网上的数据传输和提供更快的网络连接。QUIC是由Google开发的，并已被纳入标准化进程，以提高网络性能和安全性。越来越多的浏览器开始支持QUIC功能，以改进用户的网络体验。然而，某些情况下，QUIC会导致DNS泄露，因此建议禁用浏览器中的QUIC功能。

由于UDP在有些场景下必须使用真实的IP，所以 目前clash处理udp流量的域名，即使是使用fakeip模式，也一定会发起DNS请求。比如基于UDP的QUIC协议，也就是浏览器用的HTTP3，启用了QUIC的浏览器访问youtube这种支持QUIC的网站就会发起DNS请求，导致DNS泄露，解决办法是禁用浏览器的QUIC功能。
引自进阶•DNS分流篇

以Edge浏览器为例，在浏览器地址栏输入edge://flags/#enable-quic，跳转到Experimental QUIC protocol选项，将其设置为Disabled即可。

停用浏览器安全DNS

大部分浏览器现在已经默认开启“安全 DNS”功能，此功能会影响 TUN 模式劫持 DNS 请求导致反推域名失败。

以Edge浏览器为例，找到设置-隐私、搜索和服务``-安全性-使用安全的DNS指定如何查找网站的网络地址，关闭其开关即可。
